2. 用 AppID 创建/打开项目

- **环境名称**：scanbridge-prod（或任意名称）

在云开发控制台中，确认以下服务可用：
- **云托管（容器服务）**：用于部署 WebSocket 服务器

- **云函数**：用于获取云托管域名

## 3. 部署云托管（容器服务）

> 这是整个部署最关键的一步。云托管支持 WebSocket，是替代自建服务器 + Nginx + SSL 的最佳方案。

### 步骤 3.1：了解云托管原理

微信云托管（CloudBase Run）允许你以 Docker 容器方式部署服务：
- **自动分配域名**：部署完成后微信自动分配域名（`xxx.run.tcloudbase.com`）
- **自带 SSL 证书**：自动配置 HTTPS/WSS，无需手动购买证书
- **支持 WebSocket**：在服务配置中开启 WebSocket 支持
- **自动扩缩容**：根据流量自动调整实例数量（0~N，闲时可为 0）
- **按量计费**：只按实际 CPU/内存使用计费

### 步骤 3.2：进入云托管控制台

1. 在微信开发者工具中点击 **「云开发」**
2. 在控制台左侧选择 **「云托管」**
3. 如果首次使用，点击 **「开通」**

### 步骤 3.3：创建服务

点击 **「新建服务」**：
- **服务名称**：scanbridge-server
- **是否开启 WebSocket**：**必须开启**（这是关键配置！）
- **最小实例数**：1（确保服务始终可用，如果希望闲时省钱可设为 0）
- **最大实例数**：5
- **CPU 规格**：0.5核
- **内存规格**：1GB

### 步骤 3.4：配置并部署容器

有两种部署方式：

#### 方式 A：使用 CloudBase CLI 部署（推荐）

```bash
# 1. 安装 CloudBase CLI
npm install -g @cloudbase/cli

# 2. 登录
tcb login

# 3. 部署
cd server
tcb framework deploy
```

CLI 会根据 `cloudbaserc.json` 自动构建 Docker 镜像并部署。

#### 方式 B：手动在控制台上部署

1. 在云托管服务页面点击 **「新建版本」**
2. 选择 **「本地上传」** 或 **「代码仓库」**
3. 上传 `server/` 目录（包含 Dockerfile）
4. 配置端口：8080
5. 确认 **开启 WebSocket**（在高级设置中确认）
6. 点击部署

### 步骤 3.5：获取服务域名

部署完成后，在云托管服务详情页可以看到：
- **服务域名**：`https://scanbridge-xxxxx.ap-shanghai.run.tcloudbase.com`
- **WebSocket URL**：将 https 替换为 wss，即 `wss://scanbridge-xxxxx.ap-shanghai.run.tcloudbase.com`

**记录这个 URL**，后续需要填入：
1. 云函数 `getCloudHostingUrl/index.js` 中的 `HTTP_URL`
2. 桌面客户端的连接地址

### 步骤 3.6：配置自定义域名（可选）

如果你有自己的域名，可以在云托管中绑定自定义域名：
1. 在服务设置中点击 **「自定义域名」**
2. 添加你的域名（如 `scan.yourcompany.com`）
3. 按指引完成域名解析和 SSL 证书配置

## 4. 配置云数据库

### 步骤 4.1：创建集合

在云开发控制台 → **数据库** 中创建以下集合：

| 集合名 | 说明 | 安全规则 |
|--------|------|----------|
| scan_history | 扫码历史记录 | 读：true；写：`auth.openid == doc.openid`（用户只能写自己的数据） |
| pair_sessions | 配对会话（可选，当前会话数据在服务器内存中） | 读：true；写：true |

### 步骤 4.2：配置安全规则

对 `scan_history` 集合：
1. 点击集合名 → **「安全规则」**
2. 设置为：所有用户可读，创建者可写

这样扫码历史可以跨设备查看，但每个用户只能修改自己的数据。

### 步骤 4.3：添加索引（可选，提升查询性能）

对 `scan_history` 集合添加索引：
- 字段：`timestamp`，排序：降序（用于按时间排序查询）
- 字段：`openid + timestamp`，复合索引（用于用户个人历史查询）

## 5. 部署云函数

### 步骤 5.1：上传云函数

在微信开发者工具中：
1. 展开 `cloudfunctions/` 目录
2. 右键点击 `getCloudHostingUrl` 目录
3. 选择 **「上传并部署：云端安装依赖」**

### 步骤 5.2：修改云函数中的 URL

上传前，需要将云函数 `index.js` 中的 URL 替换为你实际获取的云托管域名：

```javascript
// cloudfunctions/getCloudHostingUrl/index.js
const HTTP_URL = 'https://你的实际云托管域名.ap-shanghai.run.tcloudbase.com';
const WS_URL = HTTP_URL.replace('https://', 'wss://');
```

### 步骤 5.3：测试云函数

在开发者工具中：
1. 右键 `getCloudHostingUrl` → **「云端测试」**
2. 确认返回的 `url` 字段是正确的 `wss://` 地址

## 6. 适配小程序代码

### 步骤 6.1：填入环境 ID

在 `app.js` 中修改云开发环境 ID：

```javascript
wx.cloud.init({
  env: '你的云开发环境ID',  // 例如 'scanbridge-xxxxx'
  traceUser: true,
});
```

### 步骤 6.2：填入 AppID

在 `project.config.json` 中：

```json
{
  "appid": "你的小程序AppID"
}
```

### 步骤 6.3：确认 `"cloud": true`

`app.json` 中已有 `"cloud": true`，确保这个字段存在。

### 步骤 6.4：开发者工具中勾选设置

在微信开发者工具的 **「详情」→「本地设置」** 中：
- 勾选 **「不校验合法域名」**（仅在开发调试时）
- 确认勾选 **「启用自定义处理网络请求」**

## 7. 配置域名白名单

> 云托管部署后，域名白名单配置比自建服务器简单得多。

### 步骤 7.1：获取云托管域名

从云托管控制台获取服务域名，格式如：
- `https://scanbridge-xxxxx.ap-shanghai.run.tcloudbase.com`

### 步骤 7.2：在小程序后台配置域名

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 开发管理 → 开发设置 → 服务器域名
3. 添加以下域名：

| 域名类型 | 填写内容 |
|----------|----------|
| socket 合法域名 | `wss://scanbridge-xxxxx.ap-shanghai.run.tcloudbase.com` |

> 注意：云托管的域名已经在微信白名单中，通常不需要额外添加！但为了确保 WebSocket 连接正常，建议还是明确添加 socket 合法域名。

### 步骤 7.3：开发模式下的注意事项

- 开发阶段：在开发者工具中勾选 **「不校验合法域名」**，可以使用 `ws://localhost:8080` 调试
- 真机预览：需要使用云托管的 `wss://` 地址
- 生产环境：必须使用 `wss://` 地址且域名在小程序后台白名单中

## 8. 桌面客户端配置

### 步骤 8.1：安装依赖

```bash
cd desktop-client
pip install -r requirements.txt
```

### 步骤 8.2：修改连接地址

在 `main.py` 中，将默认的 WebSocket 地址改为云托管的域名：

```python
# 在桌面客户端启动时输入，或修改默认值
DEFAULT_SERVER = "wss://scanbridge-xxxxx.ap-shanghai.run.tcloudbase.com"
```

### 步骤 8.3：启动桌面客户端

```bash
python main.py
```

启动后会显示配对二维码和 6 位配对码。

### 步骤 8.4：打包为 exe（可选）

```bash
pip install pyinstaller
pyinstaller --onefile --windowed main.py
```

生成的 `dist/main.exe` 可以直接分发给其他电脑用户。

## 11. 成本预估

### 微信云开发免费额度（基础套餐 1）

| 资源 | 免费额度 | 超出后单价 |
|------|----------|------------|
| 云数据库 | 2GB 存储 | 约 0.05 元/GB/天 |
| 云函数调用 | 5 万次/月 | 约 0.0133 元/千次 |
| 云托管 | 无免费额度 | 按量计费 |

### 云托管按量计费

| 规格 | 单价（按实际运行时间） |
|------|------------------------|
| 0.5核 1GB | 约 0.055 元/分钟 |
| 1核 2GB | 约 0.111 元/分钟 |

**预估月费用**（轻量使用场景，每天活跃约 4 小时）：
- 云托管：4h × 60min × 0.055 元 ≈ 13.2 元/天 → 约 400 元/月
- 如果设置 **最小实例数为 0**，闲时自动缩容到 0，费用大幅降低
- 云数据库：免费额度足够初期使用

> 建议：初期设置最小实例数为 **0**，闲时不产生费用。首次连接可能需要冷启动（约 2~5 秒延迟），但对扫码场景影响不大。

## 12. 常见问题

### Q1：云托管支持 WebSocket 吗？

**是的**。微信云托管支持 WebSocket，但需要在创建服务时 **明确开启 WebSocket 支持**。如果创建服务时未开启，需要在服务设置中修改。

### Q2：云托管冷启动延迟怎么处理？

设置最小实例数为 0 时，长时间无请求后服务会缩容到 0。下次连接需要冷启动（约 2~5秒）。解决方案：
- 设置最小实例数为 1（始终有 1 个实例，无冷启动，但持续产生费用）
- 在小程序中添加连接等待提示
- 利用心跳保活机制维持连接

### Q3：云函数获取域名返回错误？

确保：
1. 云函数中的 `HTTP_URL` 已替换为你实际获取的云托管域名
2. 云托管服务已成功部署
3. 云函数已正确上传并部署

### Q4：WebSocket 连接失败？

检查以下项：
1. 云托管服务是否正在运行（在控制台查看实例状态）
2. WebSocket 是否在服务设置中开启
3. URL 格式是否正确（`wss://` 而不是 `https://`）
4. 小程序后台的 socket 合法域名是否包含该域名
5. 桌面客户端连接的是 `wss://` 地址（不是 `ws://`）

### Q5：云数据库写入失败？

检查：
1. 集合是否已创建
2. 安全规则是否正确配置
3. 云开发环境 ID 是否正确
4. `store.js` 中的 `useCloudDB` 是否为 `true`

### Q6：自建服务器 vs 云托管，选哪个？

| 维度 | 自建服务器 | 微信云托管 |
|------|-----------|------------|
| 前置成本 | 需购买域名 + SSL + VPS | 无前置成本 |
| 运维成本 | 需自己维护 Nginx、pm2、SSL 续期 | 全托管，零运维 |
| WebSocket 支持 | 需手动配置 Nginx 反代 | 自动支持 |
| 域名白名单 | 需手动在小程序后台配置 | 自动包含 |
| 扩缩容 | 手动扩容 | 自动扩缩容 |
| 冷启动 | 无 | 有（设最小实例为 0 时） |
| 适合场景 | 大量实时连接、低延迟要求 | 中小规模、零运维需求 |

**建议**：初期上线选云托管，零运维快速上线。后续如果日活超千且需要更低延迟，可切换到自建服务器。

### Q7：如何从云托管切换到自建服务器？

1. 在小程序设置页面点击「切换到自建服务器」
2. 输入自建服务器地址
3. 桌面客户端切换连接地址
4. 小程序无需修改代码，自动降级到本地模式

## 部署流程一览

```
1. 注册小程序 → 获取 AppID
      ↓
2. 开发者工具 → 开通云开发 → 记录环境 ID
      ↓
3. 云托管控制台 → 创建服务 → 开启 WebSocket → 部署容器 → 获取域名
      ↓
4. 云数据库 → 创建 scan_history 集合 → 配置安全规则
      ↓
5. 云函数 → 上传 getCloudHostingUrl → 替换域名 → 部署
      ↓
6. 小程序代码 → 填入环境 ID 和 AppID → 确认 "cloud": true
      ↓
7. 小程序后台 → 配置 socket 合法域名（或确认云托管域名已自动白名单）
      ↓
8. 桌面客户端 → 修改连接地址为云托管域名 → 启动 → 配对
      ↓
9. 真机预览 → 测试完整流程 → 验证云数据库写入
      ↓
10. 上传代码 → 提交审核 → 发布上线
```
